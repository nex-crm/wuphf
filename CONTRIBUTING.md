# Contributing to WUPHF

A working agreement, not a wish list. Follow the rules; question the rules in PRs.

## The bar

Every PR earns its merge. The bar is:

1. **Clarity over cleverness.** If two readers diverge on what a function does, the function is wrong, not the readers.
2. **No new features in refactor PRs**, no refactors in feature PRs. One thing per PR.
3. **No performance regressions.** Hot paths get bench baselines (`go test -bench`); regressing is a CI failure.
4. **Delete, don't deprecate.** Private monorepo, single team. Git remembers. See [§ Dead code](#dead-code).
5. **Tests must point at one concern.** A failure must localize the bug. See [`docs/CODE-QUALITY.md`](docs/CODE-QUALITY.md) for what an "epicenter test" looks like.
6. **Type safety is load-bearing.** Wire types are codegen'd from Go to TS — never hand-edit `web/src/types/api.gen.ts`.

## Before you start

Run [`scripts/bootstrap.sh`](scripts/bootstrap.sh) to install lefthook hooks. Don't skip them.

Read [`ARCHITECTURE.md`](ARCHITECTURE.md). One page. Read it before you touch core packages.

## File size

| Range | Action |
|---|---|
| 0–800 LOC | Fine. Move on. |
| 800–1500 LOC | Warning at PR review time. Plan the next decomposition. |
| 1500+ LOC | Hard fail in CI unless the path is on `scripts/file-size-allowlist.txt`. |

The allowlist enumerates today's god files. It is **forward-only** — entries can be removed or shrunk, never added or grown. Every refactor PR description must list net LOC delta and updated allowlist diff.

The 1500 cap is calibrated against the median size of #503's `launcher_*.go` siblings + 50% headroom. If a file is over budget, the answer is decomposition, not exception.

## Decomposition patterns

Two patterns, both already proven in this repo:

- **Horizontal slice** (PR #501, `channelui` extraction): pull a pure-function layer (rendering, projection, validation) out of a stateful god file into its own package. The new package does no IO and holds no state. Use when: the layer is pure and consumed by only one binary.

- **Vertical slice** (PR #503, `launcher.go` decomposition): split a god struct's methods into themed sibling files in the same package. Naming: `<entity>_<concern>.go` (`launcher_boot.go`, `launcher_session.go`). Use when: the responsibilities share state but are separable concerns.

Pick the one that fits. Don't invent a third pattern. See [`docs/CODE-QUALITY.md`](docs/CODE-QUALITY.md) for the decision tree.

## Tests

- **Epicenter:** every test must point at exactly one production file or concern. If a failure could mean any of three things, split the test.
- **No `time.Sleep` in tests.** Use a manual clock fixture (`internal/team/scheduler.go`'s `clock` interface, or the equivalent for new packages). A test that sleeps is a test that flakes.
- **No real network or shells.** `httptest` for HTTP; fake runners for `exec.Command`. See `internal/team/tmux_runner.go` for the pattern.
- **Race detector on by default.** If `-race` reports a race, fix the race. If a package is currently exempt (today: none — `internal/teammcp` has its own carve-out for unrelated reasons), it's a debt to be paid, not a target to imitate.
- **Coverage floors are forward-only.** A package's coverage can rise; it can't fall. New code lands with tests.

## Dead code

> A function that nobody calls is debt — except when it isn't.

Before deleting a symbol, run the **three-question gauntlet**:

1. **Extra-build-graph references?** Cron yaml, GitHub Actions, shell scripts in `scripts/`, `evals/` harnesses, env-gated debug commands. `git grep <symbol>` across the *whole repo*, not just `*.go`.
2. **Error-recovery path?** Coverage tools mark `recover()` handlers, panic-cleanup, and fallback branches as dead because tests don't trigger them. **Don't delete these — write a test that does.**
3. **Reflection / string lookup?** Handler registries, command tables, plugin systems. Search for the *string form* of the symbol, not just the identifier.

Pass all three? Delete it. No `// Deprecated:` comments. No backwards-compatibility shims. Git remembers.

## Type sharing across surfaces

Wire-shaped types live in `internal/team/api/` and are the source of truth. `web/src/types/api.gen.ts` is regenerated by `make generate`. CI fails if regen produces a diff.

- **Hand-editing `*.gen.ts` is a build break.** Edit the Go struct.
- **JSON tag mismatches are silent bugs.** When changing a field, regenerate.
- **Adding optional fields is non-breaking only if the JSON omits them when zero.** Use `omitempty` deliberately.

## TUI vs web surface coverage

We don't aim for parity. We aim for **intentional asymmetry**.

See [`docs/surfaces.md`](docs/surfaces.md) for the canonical feature × surface matrix. If you're adding a feature on one surface, decide *and document* whether it should exist on the other. Not every feature should — some are pixel-shaped, some are terminal-shaped.

## Commits and PRs

- Conventional Commits enforced by commitlint. Allowed types: `build`, `chore`, `ci`, `docs`, `feat`, `fix`, `perf`, `refactor`, `revert`, `style`, `test`. Free-form scopes (`fix(team):`, `feat(web):`).
- Body lines ≤ 100 chars.
- Always branch + draft PR. Never push to `main`.
- PR description must include: net LOC delta, file-size allowlist diff (if applicable), bug-hunt findings (refactor PRs).
- Run `bash scripts/test-go.sh` and the web build before marking ready.
- Never `--no-verify`. If a hook fails, the hook is doing its job.

## Refactor expectations

A decomposition PR that doesn't surface bugs is a decomposition PR that didn't read the code carefully enough. PR #501 and #503 each found multiple criticals during the split (cycle bugs, goroutine leaks, race conditions, untyped context keys). Expect to find some, and document what you fixed in the PR description under a **Bug-hunt** section.

## When to ignore these rules

When the rule produces an obviously wrong outcome — and you can articulate why in the PR description. The rules are calibrations, not commandments. But "I disagree with the rule" without a written argument is not a reason; it's noise.
