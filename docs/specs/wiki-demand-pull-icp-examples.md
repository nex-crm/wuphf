# Demand-Pull Synthesis — ICP Tutorial Examples

These three examples are the spec for `feat/wiki-demand-pull` (PR 1).
The feature is done when all three work end-to-end, in order.

**What demand-pull means:** In `Demand` mode (`WUPHF_ENTITY_SYNTHESIS_MODE=demand`),
the broker never auto-synthesizes entity briefs on the fact-ingestion path.
Instead, synthesis fires the first time `BuildArticle` is called on a ghost
brief that has enough facts to synthesize from. The caller gets the ghost
immediately (fast), and the real brief commits in the background.

---

## Example 1 — Alex Chen, Account Executive (happy path)

**Persona:** Alex runs outbound at a 40-person SaaS company. The team ingests
emails and Slack into WUPHF. `WUPHF_ENTITY_SYNTHESIS_MODE=demand` is set.

**Setup:** Acme Corp just booked a discovery call. Over the past two weeks,
four emails and seven Slack messages have been extracted, producing 11 facts
in `wiki/facts/company/acme-corp.jsonl`. The ghost brief exists at
`team/company/acme-corp.md` with `ghost: true` in frontmatter.
No synthesis has run yet (demand mode: it never fires on ingest alone).

**Step 1 — Alex opens the article (T=0)**

```go
BuildArticle(ctx, "team/company/acme-corp.md", "web", readLog)
```

Broker detects `ghost: true`. Fact log has 11 entries ≥ `DefaultSynthesisThreshold`
(5). Calls `synth.EnqueueSynthesis("company", "acme-corp", ArchivistAuthor)`.

Returns immediately with:

```json
{
  "title": "Acme Corp",
  "content": "---\nslug: acme-corp\n...\nghost: true\n---\n\n# Acme Corp\n\n## Signals\n\n_No facts synthesized yet._\n\n---\n*Generated by archivist.*",
  "ghost": true,
  "synthesis_queued": true,
  "human_read_count": 1,
  "days_unread": 0
}
```

**What Alex sees:** The article opens instantly showing the ghost stub. A
spinning "Generating brief..." indicator appears below the title (keyed on
`synthesis_queued: true`). The article is readable and not blank — it has the
entity name and a Signals section header.

**Step 2 — Synthesis completes (T=~10s)**

The synthesizer runs the LLM call, commits the real brief to
`team/company/acme-corp.md` (removing `ghost: true` from frontmatter), and
removes the job from the in-flight coalescing set.

**Step 3 — Alex reloads or the UI polls (T=~12s)**

```go
BuildArticle(ctx, "team/company/acme-corp.md", "web", readLog)
```

Broker reads the file. No `ghost: true` in frontmatter. Returns:

```json
{
  "title": "Acme Corp",
  "content": "---\nslug: acme-corp\n...\n---\n\n# Acme Corp\n\n## Summary\n\nAcme Corp is evaluating a migration from their legacy CRM...\n\n## Key Contacts\n\n- **Dana Liu** (VP Sales) — primary champion...\n\n## Last Touchpoints\n\n- 2026-04-29: Discovery call booked via LinkedIn outreach...",
  "ghost": false,
  "synthesis_queued": false,
  "human_read_count": 2,
  "days_unread": 0
}
```

**What Alex sees:** The brief fills in with real content. The spinner
disappears. Alex has the account context she needed before her call.

**Assert:** `ghost: false`, `synthesis_queued: false`, content is non-stub.

## Example 2 — Jordan Park, Customer Success Manager (MCP agent path)

**Persona:** Jordan's team runs a `prep-qbr` Nex agent that pulls wiki briefs
for accounts before quarterly business reviews. The agent calls the broker MCP
tool `wiki_read_article`. `WUPHF_ENTITY_SYNTHESIS_MODE=demand` is set.

**Setup:** TechFlow Inc has a ghost brief at `team/company/techflow-inc.md`
with 8 facts in its fact log. The agent runs via `NEX_AGENT_SLUG=prep-qbr`.

**Step 1 — Agent requests the article**

MCP tool call → broker hits:

```go
BuildArticle(ctx, "team/company/techflow-inc.md", "prep-qbr", readLog)
```

8 facts ≥ threshold. Synthesis enqueued. Returns:

```json
{
  "title": "TechFlow Inc",
  "content": "---\nslug: techflow-inc\n...\nghost: true\n---\n\n# TechFlow Inc\n\n## Signals\n\n_No facts synthesized yet._",
  "ghost": true,
  "synthesis_queued": true,
  "agent_read_count": 1,
  "human_read_count": 0,
  "days_unread": 0
}
```

**What the agent sees:** JSON response with `ghost: true, synthesis_queued: true`.
The agent writes to its notebook: "TechFlow brief is generating. Will re-check
in 15 seconds." (Agent-side behavior — not part of this PR.)

**Step 2 — Agent re-requests after synthesis completes**

```go
BuildArticle(ctx, "team/company/techflow-inc.md", "prep-qbr", readLog)
```

Returns real brief:

```json
{
  "title": "TechFlow Inc",
  "ghost": false,
  "synthesis_queued": false,
  "agent_read_count": 2,
  "content": "# TechFlow Inc\n\n## Summary\n\nTechFlow is on a month-to-month contract expiring 2026-06-30..."
}
```

**What Jordan sees (via agent output):** The QBR prep doc includes the full
account brief. Renewal risk, open tickets, and key contacts are all present.

**Assert:** Second call returns `ghost: false`. `agent_read_count` increments
on both calls — ghost reads still count for staleness tracking.

## Example 3 — Dana Okonkwo, RevOps Lead (below-threshold ghost, no queue)

**Persona:** Dana runs a RevOps audit script that opens every entity brief to
populate a spreadsheet. One prospect — CloudVault Inc — has a ghost brief but
only 2 facts (below `DefaultSynthesisThreshold` of 5). Not enough signal to
synthesize anything meaningful.

**Setup:** `team/company/cloudvault-inc.md` exists with `ghost: true`.
Fact log has 2 entries. `WUPHF_ENTITY_SYNTHESIS_MODE=demand` is set.

**Step 1 — Script calls BuildArticle**

```go
BuildArticle(ctx, "team/company/cloudvault-inc.md", "web", readLog)
```

Broker detects `ghost: true`. Fact log has 2 entries < threshold (5).
**No synthesis enqueued.** Returns ghost as-is:

```json
{
  "title": "CloudVault Inc",
  "content": "---\nslug: cloudvault-inc\n...\nghost: true\n---\n\n# CloudVault Inc\n\n## Signals\n\n_No facts synthesized yet._",
  "ghost": true,
  "synthesis_queued": false,
  "human_read_count": 1,
  "days_unread": 0
}
```

**What Dana sees:** The spreadsheet row for CloudVault shows "Ghost — 2 facts,
not enough to synthesize." No spinner. No LLM call fired. The brief stays as
a ghost until more facts arrive and someone opens it again.

**Assert:** `synthesis_queued: false` when fact count < threshold.
No `EnqueueSynthesis` call made. No LLM usage incurred.

## Field contract added by this PR

Two new fields on `ArticleMeta` (Go) / `WikiArticle` (TypeScript):

| Field | Type | Source |
|---|---|---|
| `ghost` | `bool` | Parsed from frontmatter `ghost: true` at serve time |
| `synthesis_queued` | `bool` | Computed from the synthesizer's in-flight coalescing set — **never persisted** |

`synthesis_queued: true` is only possible when `ghost: true`. A real brief
(`ghost: false`) never shows `synthesis_queued: true`.
